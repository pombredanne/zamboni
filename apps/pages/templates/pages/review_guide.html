{{ _('Review Guidelines') }}

{% trans %} Add-on reviews are a way for add-on site users to share their opinions about the add-ons that they have installed and used. Editors reserve the right to refuse or remove any review that does not comply with these guidelines. {% endtrans %}

{{ _('Some tips for writing a great review') }}

{{ _('Do:') }}

  • {% trans %} Write like you are telling a friend about your experience with the add-on. {% endtrans %}
  • {{ _('Keep reviews concise and easy to understand.') }}
  • {{ _('Give specific and helpful details. For example:') }}
    • {{ _('Did the add-on work as you expected it to?') }}
    • {{ _('What features did you like or dislike?') }}
    • {{ _('Was it useful?') }}
    • {{ _('Was it easy to use?') }}
    • {{ _('Will you continue to use this add-on?') }}
  • {% trans %} Take a moment to read your review before submitting it to avoid embarrassing typos or grammatical errors. {% endtrans %}
  • {% trans %} Avoid referring to topical or temporal references since reviews can remain on the site for a long time. {% endtrans %}

{{ _('Don’t:') }}

  • {% trans %} Submit simple reviews such as "Great!", "wonderful", or "bad" with no further explanation. {% endtrans %}
  • {% trans %} Post bug or problem reports in reviews. Use the available support options for each add-on. {% endtrans %}
  • {% trans %} Write reviews for add-ons which you have not personally used. {% endtrans %}
  • {% trans %} Use profanity, sexual language or language that can be construed as hateful. {% endtrans %}
  • {% trans %} Include HTML, links to malware, source code or code snippets. Reviews are meant to be text only. {% endtrans %}
  • {% trans %} Make false statements, disparage add-on authors or personally insult them. {% endtrans %}
  • {% trans %} Use reviews as a way to request support for a specific Firefox (or other application) version. {% endtrans %}
  • {% trans %} Include your own email, phone number, or other personal details. {% endtrans %}
  • {% trans %} Post reviews for an add-on you or your organization wrote or represent. {% endtrans %}

{{ _('Guidelines for rating add-ons') }}

{% trans %} Add-ons ratings should be fair and give a good indication of overall quality and usefulness. Don't just give it 5 stars if you like it or 1 star if you hate it. Consistent grading is an important part of any review. {% endtrans %}

  • {% trans %} 5 stars: Excellent. Not only does everything it claims, but it is significantly more useful than many other add-ons. This add-on is likely to be one you use often and you highly recommend it. {% endtrans %}
  • {% trans %} 4 stars: Good. Useful and easy to use, though not necessarily special. This add-on is recommended but may be more suited for some people over others. {% endtrans %}
  • {% trans %} 3 stars: Passable. May have some design issues and/or some missing features. Any problems it has are not drastic enough to recommend against, and some people may still find it useful. Recommended for those who need it and are willing to give it a try. {% endtrans %}
  • {% trans %} 2 stars: Poor. Add-on is of low quality, usability, or just doesn't do what it says it should. You are recommending against this add-on, while noting that it might have some redeeming value for a few people. {% endtrans %}
  • {% trans %} 1 star: Bad. Add-on either doesn't work correctly or is utterly useless. You can't think of any significant reason why someone should give it a try and are recommending that it be avoided completely. {% endtrans %}

{% trans %} There's nothing wrong with giving an add-on a perfect or horrible review, just please indicate why you are doing so. The stars mean nothing if you don't tell us why. {% endtrans %}

{{ _('Frequently Asked Questions about Reviews') }}

{{ _('How can I report a problematic review?') }}

{% trans %} Please report or flag any questionable reviews by clicking the "Report this review" and it will be submitted to the site for moderation. Editors will use the Review Guidelines to evaluate whether or not to delete the review or restore it back to the site. {% endtrans %}

{{ _('I’m an add-on author, can I respond to reviews?') }}

{% trans %} Yes, add-on authors can provide a single response to a review. Additional discussions or followup should be moved to a support forum or discussion group. {% endtrans %}

{{ _('I’m an add-on author, can I delete unfavorable reviews or ratings?') }}

{% trans %} In general, no. But if the review did not meet the review guidelines outlined above, you can click "Report this review" and have it moderated by an editor. If a review included a complaint that is no longer valid due to a new release of your add-on, we may consider deleting the review. Submit your detailed request to amo-editors@mozilla.org. {% endtrans %}
